# 支付配置速查（前端静态展示版）

这份文档只解决两件事：
1. 你要在哪里填配置。
2. 每个配置项填什么。

## 1) 在哪里填

如果你部署在 Vercel：
1. 打开项目后台。
2. 进入 Settings -> Environment Variables。
3. 按下面变量名逐条新增。
4. 选中环境：Production（至少）。
5. 保存后重新部署一次。

本地调试可在项目根目录创建 .env.local（不提交到仓库）。

## 2) 当前版本配置说明

当前代码已改为纯前端静态收款展示，支付区域直接使用固定二维码：

- https://i.postimg.cc/P525Vk7w/IMG-9071.png

因此：
1. 不依赖后端支付接口。
2. 不依赖数据库。
3. 不依赖验签/回调/订单状态查询。

## 3) 平台侧无需额外配置

本版本无需配置支付宝开放平台通知地址，也无需服务端支付参数。

## 4) 上线前 2 分钟自检

1. 弹窗内显示的是你的支付宝二维码。
2. 点击“我已完成支付，继续解牌”后可直接进入进阶解牌。
3. 页面刷新后 24 小时内仍保持进阶可用（本地缓存 token）。

## 5) 当前项目支付结论

当前项目已切换为“静态二维码 + 前端继续”模式。

特点：
1. 交互最简单。
2. 不需要维护支付后端链路。
3. 适合你当前 offline/小范围使用场景。

## 6) 常见问题

1. 二维码没显示
- 检查图片链接是否可公网访问。

2. 点击后没继续解牌
- 刷新页面后重试；若无痕模式可能拦截本地缓存。

3. 想替换成你自己的二维码
- 直接把代码里的二维码地址替换成你的图片地址。
